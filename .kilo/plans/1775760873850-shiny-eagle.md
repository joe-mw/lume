# Lume Architecture Plan

## 1. Overview
Lume is a cross-platform (iOS, tvOS, macOS) IPTV player built with SwiftUI, SwiftData, and KSPlayer. The architecture focuses on offline-first capabilities via content indexing, clean UI/UX, and robust playback of Xtream Codes API content.

## 2. Core Technologies
- **UI Framework:** SwiftUI (following modern design patterns like liquid glass and minimalist tab bars).
- **Local Storage:** SwiftData for content indexing (playlists, categories, streams, movies, series, watch history).
- **Networking:** `URLSession` wrapped in an `XtreamClient` to interface with the Xtream API.
- **Media Player:** KSPlayer (with a modular design to support future AVPlayer/VLC fallbacks).

## 3. Architecture Pattern
We will use a **Model-View-Service** (or modern SwiftUI MV) architecture:
- **Views:** Directly observe local state using `@Query` for SwiftData models, minimizing boilerplate.
- **Services/Managers:** Handle complex business logic, networking, and syncing. This keeps views lightweight.

## 4. Data Models (SwiftData)
- `Playlist`: Stores Xtream API credentials (URL, username, password), server info, and user status.
- `Category`: Represents Live TV, VOD, or Series categories. Associated with a `Playlist`.
- `MediaItem`: A base model or protocol for `LiveStream`, `Movie`, and `Series`.
- `Episode`: Belongs to a `Series`.
- `EPGListing`: EPG data associated with a `LiveStream`.
- *Note:* SwiftData will serve as the single source of truth for the UI.

## 5. Services & Managers
- `XtreamClient`: Handles raw API requests (parsing JSON to DTOs) as defined in `XtreamAPI.md`.
- `SyncManager`: Coordinates between `XtreamClient` and SwiftData. It fetches the latest data, reconciles differences, and saves to the local database for offline access.
- `PlayerManager`: Manages the state of KSPlayer, handles URL construction for streams, aspect ratio, subtitles, and tracks watch progression.
- `SettingsManager`: Manages user preferences (AppStorage/UserDefaults) such as dark mode, player settings, and UI customization.

## 6. UI Navigation Structure
- `MainTabView`: Custom tab bar that shrinks on scroll.
- `HomeView`: Recommendations, recently watched, trending (Future).
- `LiveTVView`: Category list expanding to channel grids with EPG data.
- `MoviesView` / `SeriesView`: Horizontal scrolling rows of posters.
- `DetailView`: Shared layout for Movies and Series (poster, metadata, play button, seasons/episodes picker).
- `PlayerView`: The immersive playback experience.
- `SettingsView`: Playlist management and app preferences.

## 7. Tradeoffs & Decisions
1. **State Management:** Modern SwiftUI approach (Views + Services with `@Query`) will be used to reduce boilerplate while maintaining separation of concerns for complex logic.
2. **Background Sync:** Initial implementation will focus on syncing content only when the app is active to simplify the initial release. BackgroundTasks can be explored later.
3. **Player Modularization:** KSPlayer will be integrated directly to ship the prototype faster. An abstract `PlayerEngine` protocol can be refactored in the future when adding AVPlayer/VLC support.

## 8. Next Steps
Once the architectural tradeoffs are decided, we will begin with scaffolding the SwiftData models and the Xtream API networking layer.
